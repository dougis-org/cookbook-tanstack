## Why

The TanStack Router/Start ecosystem (React Router, React Start, router-plugin, devtools, SSR-query) and React Query packages have drifted behind the installed versions. Keeping these in sync with upstream reduces exposure to known bugs, captures scroll-restoration and HMR fixes shipped since 1.159.5, and keeps the dependency graph coherent (all TanStack Router packages share internal APIs and must stay at matching versions).

## What Changes

- `@tanstack/react-router` 1.159.5 → 1.168.10
- `@tanstack/react-start` 1.159.5 → 1.167.16
- `@tanstack/router-plugin` 1.159.5 → 1.167.12
- `@tanstack/react-router-devtools` 1.159.5 → 1.166.11
- `@tanstack/react-router-ssr-query` 1.159.5 → 1.166.10
- `@tanstack/react-query` 5.90.21 → 5.96.2
- `package.json` version specifiers updated to reflect the new baseline

No public APIs used by this codebase are removed or changed. The one internal breaking change (`pendingMatches`/`cachedMatches` removed from `RouterState` in 1.167.0) does not affect this codebase — `router.state` is never accessed directly here.

## Capabilities

### New Capabilities

None — this is a dependency maintenance change with no new product capabilities.

### Modified Capabilities

None — no spec-level behavior changes. The `unsaved-changes-guard` spec is unaffected: `useBlocker` public API is unchanged, though manual smoke-testing is warranted (see Risks).

## Impact

**Files touched:**
- `package.json` — version specifiers updated
- `package-lock.json` — regenerated by npm

**Runtime behavior potentially affected:**
- Scroll restoration timing (throttling removed in 1.168.5) — used via `scrollRestoration: true` in `src/router.tsx`
- React HMR for inline component definitions (1.166.8) — dev-only improvement

**Test surface:**
- `src/components/recipes/RecipeForm.tsx` — `useBlocker` usage; underlying pending-state machinery was refactored in 1.167.0
- All routes — general regression risk

## Problem Space

**In scope:**
- Bumping the six TanStack packages listed above to their current latest versions
- Verifying build, unit tests, and E2E tests pass after the bump

**Out of scope:**
- Adopting any new APIs introduced in the updated versions (e.g., `staleReloadMode`)
- Updating other non-TanStack dependencies
- Any feature work

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| `useBlocker` behavior change due to pending-state refactor (1.167.0) | Low — public API unchanged | Manual + E2E test of "navigate away with unsaved changes" in RecipeForm |
| Scroll restoration timing change (throttling removed, 1.168.5) | Low — it's a fix | Verify scroll position after navigation in E2E run |
| Unforeseen type errors from minor API shape changes | Low | `npm run build` catches these |
| Test suite exposes regressions | Medium — broad change | Full `npm run test` + `npm run test:e2e` required before merging |

## Non-Goals

- This change does not introduce new features or refactor existing code.
- No changes to application logic, routes, components, or styles.
- Does not address any other open dependency issues.

## Open Questions

No unresolved ambiguity. The upgrade path is clear: install, build, test. Breaking-change research (conducted during exploration) found no blockers for this codebase.
