# Tests

## Overview

This document outlines the tests for the `refactor-secure-test-helper-random` change. All work should follow a strict TDD (Test-Driven Development) process where feasible, focusing on ensuring the new secure suffix generator works as expected and that existing E2E tests remain functional.

## Testing Steps

For each task in `tasks.md`:

1.  **Write a failing test:** For the new utility, write a unit test that fails if the utility is missing or doesn't meet requirements.
2.  **Write code to pass the test:** Implement the utility or update the helpers.
3.  **Refactor:** Improve the code quality and structure while ensuring the tests still pass.

## Test Cases

- [ ] **Test Case 1 for Step 3 (Utility):** Create `src/e2e/helpers/__tests__/utils.test.ts` (if a vitest-compatible location exists) and assert that `getUniqueSuffix()` returns a string with a timestamp and 8-character random part.
- [ ] **Test Case 2 for Step 3 (Uniqueness):** Call `getUniqueSuffix()` 100 times in a loop and assert that all returned values are unique.
- [ ] **Test Case 1 for Step 4-6 (Integration):** Run existing E2E tests that rely on these helpers (e.g., `src/e2e/auth.spec.ts`, `src/e2e/recipes.spec.ts`, `src/e2e/cookbooks.spec.ts`).
- [ ] **Test Case 2 for Step 4-6 (Regressions):** Ensure that resource names generated by the new utility do not cause UI truncation or validation errors in the app.
- [ ] **Test Case 1 for Step 7 (Verification):** Run a grep search for `Math.random()` in `src/e2e/helpers/` and confirm zero matches.
